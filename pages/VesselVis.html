Felix Merkl | Vessel Visualizations

State of the art vessel visualizations

Visualizations for augmented objects are a well researched field to support medical doctors during laprascopic surgeries. In this kind of surgery, three small incisions are made into the patient's body and a camera and instruments are inserted. This camera captures images of the area of interest and is the only way for the surgeons to see the operation site.


However, the view is very limited, it is very hard to percieve the depth, and blood vessels can be covered by fatty tissue, or organs. This is very dangerous, as accidentally hurting a vessel might cause serious injuries. Therefore, augmented reality can be used to reveal vulnerable structures.


Further, different visualization techniques can be applied to the augmented objects to deal with perception problems. Thus, the main goal is to improve spatial relations and depth and shape perception of an object by encoding different perception cues into outstanding features in shading options or additional augmentations. In general, the features can be categorized into three groups: transparency, illustrative, and auxilliary augmentations.


Preim et al. [1] evaluate state of the art visualization methods. As preparation for own research, a selection of popular methods was implemented. This page shows the chosen methods and explains the most important aspects. You can test the visualization methods in a WebGL application here.

Basic Shaded Model

This is the standard rendering method for 3D models. It shades the object in a solid, non-transparent color and sets specular highlights from a virtual light source. This method can be used for medical augmented reality, however, it does not add any cues for depth and shape perception.

wire frame model

The wire frame technique does not render the faces of the 3D model, but only the edges - the connections of the model's vertices. This allows to see through the model without changing transparency values and thus see occluded parts. It is also a very common rendering technique that is pre-implemented by most platforms. However, it does not work for WebGL applications created with Unity, so you cannot find it in the demo.

Fresnel

With visualization methods in the transparency category, the alpha value of the model is changed. This causes to blend the vessel's color with the color of the background. This helps with occlusion problems and depth perception. The transparency of the object is dependent on the normal vectors and the camera perspective.

With a Fresnel effect, the alpha value changes for each point on the vessel. This alpha value is dependent on the normal vector and the view vector of the camera. As a result, the shape and the outline of the vessel are emphasized.


Ostendorf et al. [2] investigated the Fresnel effect and similar methods.

Fog

The fog effect changes the alpha value based on the distance to the camera (image depth). As important parts are mostly in the center and in the foreground of the image, this effect focuses on interesting parts and partly excludes unnecessary areas.

Chromadepth

In illustrative methods, mostly the color of the object is changed. The chromadepth technique computes the depth of the images and maps the depth value to a color. This change of color gives the viewer an easy understanding of the depth of the object.

Outline

Illustrative methods do not always add to the objects. As sometimes, the objects do not add much to the scene, a simplistic view might be better suited. The outline only shows the silhouette and removes any features.

Hatching

As the outline might be too simple, it is possible to retain some features to help with the shape perception. The features can be drawn as small hatches that resemble hand drawn shadows and are dependent on the camera view. That means for each perspective different strokes are drawn that emphasize difficult shapes.


This implementation is inspired by the work from Ritter et al. [3].

Shadow Projection

All before mentioned methods only change features of the vessel itself. However with auxilliary augmentations, it is possible to add more virtual objects into the augmentation. For example, two walls can be used as projection planes for a virtual shadow of the object. These extra perspectives onto the model can help to further understand its shape and depth.


This implementation is inspired by the work from Lawonn et al. [4].

Combination is key

Each of the visualization can help to improve the perception of augmented objects. However, they each focus on either depth or shape perception. Therefore, it is common practice to combine two or more visualization techniques. For example hatching with chromadepth or projections with chromadepth.

Dynamic vessel visualizations

The state-of-the-art techniques cover many different aspects for depth and shape cues. However, they do not utalize motion cues. The change of perspective gives a good understanding of the objects' shape and effects like motion parallax help to perceive depth.


The results of a study show a good performance for the dynamic methods but not significantly better then most state-of-the-art techniques. The participants especially favoured them in subjective questionnaires.


Naturally, the dynamic aspect is not limited to blood vessels, however, the object flow methods works best on tubular structures. Therefore, blood vessels are the perfect application. You can test the visualization methods in a WebGL application here.

Object Flow

The object flow method is a dynamic illustrative visualization technique that propagates auxilliary objects through an object along a set path. This path can be the centerline (skeleton) of the object and can also be displayed as additional perception cue. The flowing objects should indicate the direction and the distance. We achieve this by using different colors for the front and the back. As distant objects appear smaller then near objects, the distance can be estimated by comparing the sizes. However, it is also possible to add chromadepth to distinguish the depth.

Wave Propagation

The wave propagation method is a dynamic illustrative visualization method that animates view-dependent feature lines (grid lines) combined with chromadepth. The feature lines use a trail to cover a larger area. By moving the waves along the view direction at a constant speed, every part of the vessel will be emphasized. Chromadepth provides cues for depth perception.

Try the Demo

You can test the mentioned visualization methods in a WebGL application created with Unity in the browser. You can change between the methods and rotate the vessel to take a better look from other perspectives.
